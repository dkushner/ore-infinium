    Ore Infinium

    What is it?

    Ore Infinium is a game that is greatly inspired by Terraria, but of course aims to be a hell of a lot more. One way to describe it is a 2D Dungeon Crawler/Sandbox, with also an emphasis on survival. You can mine blocks, minerals, build houses, caves, and all sorts of neato things.
    Community is a huge part of development behind this project (hence why it's on github in the first place. Feel free to poke the code, make some art, and fiddle with what you can find ;)

    Why is it?

    It is a project that is open source from the very beginning, as improving upon the neat ideas that Terraria introduced to the world when it released, taking input and help from the community, and making our ideas a reality.

    It is written in SDL (just for audio, input and OpenGL context), OpenGL and C++, so is cross-platform and very fast (I use all hardware accelerated rendering)

    • It will have modding capabilities

    • It will have loads of features that I missed and I'm sure some others have as well.

    • It's an ongoing project, not an end goal. And in the event that I get hit by a bus tomorrow, feel free to fork it and pick it up!

    • Cross-platform (Windows, Linux, Mac OS X, with long-term plans for Android)

    • Many, many features are planned that people only dreamed of for Terraria to have.

    • Community driven. I'm very open to code reviews, patches, and constructive criticism, bug and feature requests, etc.

    What license is it under?

    All assets are required (by me ;) to be under a Free and Open license. The code is presently under GPLv2+ as that seems to be the licence I like the most.

    What platforms will it run on?

    The goal initially is to make it run on Linux/Unixen, Mac OS X and Windows.
    I don't want to force anyone onto a platform just to play our game.
    I would love to see it run on Android for instance. But for this to happen I'd need to run through all of my code base and make sure it runs just fine on OpenGL ES 2.0. Not to mention compiles and all my dependencies work on it.
    In theory it should work just fine, and just require a bit of OpenGL tweaking for it (an official port with a proper Android interface notwithstanding).
    I'll worry about getting more into porting to Android after I get something usable on the desktop. Patches welcome ;-)

    Current Status

    The project is very, very preliminary right now. If there was a word for pre-alpha, I would use it to describe this.
    But of course, it's progressing quite quickly over just the few weeks it has existed

    • I did have a working tilemap rendering fragment shader..which uses some neat tricks to render it all in super fast manner.
    • And as I'm typing this, I'm researching on how to further revamp the tilemap renderer, and implement the background (underground, and walls), in addition to making the foreground tiles meshable (cleanly, seamless).
      The design I'm planning on allows for *a lot* more flexibility than Terraria had. It means that not only is it fast and efficient on hardware, which gives me breathing room to add more features whereever I can, but tiles blend together very nicely based on neighboring tiles. This means that Worlds can be totally different in design, but still look great.

    How do I install it?

    Read the INSTALL file and it will show all the dependencies I use and where to get them, and hopefully how to install them ;-)
    After this is installed and you have followed their directions, to get my code, you can either click download the zip, or run `git clone git://github.com/sreich/ore-infinium.git`
    After you have my code, go to the top directory of it, `mkdir build; cd build;` and now to actually build it

    System Requirements

    • SDL 2.x (yet unreleased) (and SDL_mixer)

    • Platform of Windows, Linux/Unix, Mac OS X

    • Minimum RAM of 250 MiB. Ore Infinium uses so far.. an average of 60 MiB RES.
      This will change with world size, since we don't have chunk based loading (yet?), Currently the size of the Block class is 2 bytes (2 unsigned chars). Which means for a 8400x2600 map (the default, soft minimum), (8400 \* 2600 \* 2 bytes = ~44 MiB).
      Presently, to run the game it's about 60MiB of RAM usage on average. Subject to change, of course.

    • GPU and drivers capable of OpenGL 3.2+ (and hopefully OpenGL ES 2.0).
      In other words, if your GPU/drivers say DirectX 10+ capable, or you can run DirectX 10+ games then that will work.

    Planned Features

    Some of these plans are merely thoughts and things to experiment with, some are must haves and others are things that will hopefully come later.
    Of course, nothing is in stone.

    • Multiplayer (obviously, a must ;)

    • Dungeons

    • Ladders! Many games make it cumbersome to come up from your mining pit after you're done, often you end up dying because you slipped. Also, portals help fix that ;)

    • Villages! (randomly generated). Think similar to Minecraft

    • A nice tutorial system, or at least something brief to stay out of the player's way, but also give them help when they need it.

    • NPCs/merchants you can "place" in your home.

    • Teleporters/Portals! Seriously. This is a definite todo in my mind.

    • Crafting system more simlar to Minecraft? Would people like that? Personally, I was not much of a fan for Terraria's crafting.

    • Menu system (HUD) that is easy to navigate. A lot of games suffer from that issue once they introduce crafting. The GUI/HUD will also be customizable/fully moddable, by YAML, which is a simple text format, and images.

    • Full modding support (another must). This will have to be a later feature though of course, until I get a good amount of content into the game.

    • Android and in general, a mobile port? This is very much long term though. Also, I don't have a tablet.

    • Dynamic 2D lighting and shadow system, versus Terraria's very simplistic tile-based lighting.

    • Will use some of the very useful outlets of independent music developers, and have a big selection of in-game music, just so that you don't get sick of the soundtracks while building that awesome castle of yours.

    • Water...and I'd also like to make it pressure based, but that's going to be a lot more work as well

    • Weather system - Rain, Snow
